Database

The dataset for this task consists of more than 500 hours of untrimmed videos from ActivityNet Release 1.3. There is a total of ~ 20K videos distributed among 200 activities. The distribution among training, validation and testing is 50%, 25%, and 25% respectively. The annotations for ActivityNet Release 1.3 can be obtained from the download page. Additionally, training and validation subsets will be available on March 23 2016, and testing datad without ground-truth annotations will be released on April 1 2016.

Challenge Tasks

Untrimmed video classification

This task is intended to evaluate the capability of algorithms to predict activities in untrimmed video sequences. Here, videos can contain more than one activity, and typically large time lapses of the video are not related with any activity of interest.

Activity Detection

This task is intended to evaluate the capability of algorithms to temporally localize activities in untrimmed video sequences. Here, videos can contain more than one activity instance, and mutiple activity categories can appear in the video.

Challenge Guideline

To enter the competition, first you need to create an account on Evaluation Tab. From your account you will be able to participate in ActivityNet Challenge 2016. We have a limit of five submissions per each day (maximum). For final results, only ones that were submitted before the challenge deadline and posted to the leaderboard will be considered to be participating in this challenge. Uploading JSON files using an invalid format for either classification or detection will prompt an error from our server. After our server processes your submission, a file with your results will appear for download. Additionally, you will be able to compare your results to the state-of-the-art on the Leaderboard tab.

You have to bear in mind the following format for each submission process:

  •   Classification
  •  Detection

Untrimmed video classification

Please format your results as illustrated in the example below. You can also download this example classification submission file.

                              {
version: "VERSION 1.2",
    results: {
          5n7NCViB5TU: [
              {
              label: "Discus throw", # At least one prediction per video is required.
              score: 1
              },
              {
              label: "Shot put",
              score: 0.777
              }
          ]
      }
}

Activity Detection

Please format your results as illustrated in the example below. You can also download this example detection submission file.

{
  version: "VERSION 1.2",
      results: {
          5n7NCViB5TU: [
            {
            label: ["Discus throw", "Shot put", "Brushing teeth"], # Maximum 3 predictions
            score: [0.64, 0.19, 0.16], # The length of this array must match the length of predicted labels
            segment: [24.25,38.08]
            },
            {
            label: ["Shot put"],
            score: [0.77],
            segment: [11.25, 19.37]
            }
      ]
 }

Requirements

For each submission, you have to attach the following files:

  • Submission Json File for each challenge task that you choose.
  • Paper or document that you shall describe us your model. This document must be attached in PDF format.

Metrics

Two different metrics are used to evaluate the performance of classification algorithms in this task:

  • Mean Average Precision (mAP): Interpolated Average Precision (AP) is used as one of the metrics for evaluating the results on each activity category. The AP is averaged over all the activity categories. For more information on Interpolated Average Precision, please read: Evaluation of ranked retrieval results.
  • Top-k classification accuracy is computed as the fraction of test videos for which the correct label is among the top-k most confident algorithm predictions. Here, the classification accuracy is computed with k=3.
